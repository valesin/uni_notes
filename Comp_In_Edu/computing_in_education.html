There are two approaches relating to computing in education: Constructivism, proposed by Piaget and Constructionism, developed by Papert.

As described in (??, ????), Constructivism revolves around the idea of experience and active interaction as necessary tools for learning.

Papert, who worked with Piaget around 1960, built upon that idea and came up with a theory that shared the importance of learning through experience and added that this process happens through the creation and sharing of a public entity. (??, a)

An important part of the philosophy of Constructionism is the concept of microworlds, described as “…subset[s] of reality or a constructed reality so … as to allow a human learner to exercise particular powerful ideas or intellectual skills” (Papert 1980, p. 204).

Papert also believed that the right tool to implement microworlds is a computer, because "its essence is its universality, its power to simulate.". It can help to create an atmosphere that supports active involvement and promotes learning.

As a consequence, he developed Logo - a functional programming language based on Lisp - with the intent of introducing children to programming from a young age.

A key feature of the Logo programming language is turtle graphics, consisting of a cursor over a Cartesian plane. The cursor has a location, a direction and a pen. The direction of the cursor can be changed by the student with commands relative to its current position, by specifying an angle of rotation. The activity consists of programming a "kangaroo" to get out of a maze by using programmatic steps.
The original activiuty was made in MAZE language but I used python, preferring an approach that more closely resembles real-life code
I also simplified the structure of the mazes, building discrete lattices instead of continuos mazes. This way, a turn can only be of 90 degrees

Keeping concepts simple by creating a discrete latex with easy way to rotate (no degrees involved) as that's not the goal and it would add useless friction. We want to concentrare only on the computational thinking aspect

The fact that every command relates to the kangaroo allows body-syntonic learning as theorized by papert for his programming language LOGO.

J Wing position is that CT is composed of abstraction and automation

She thinks that „CT is a method of solving a problem that focuses on its description, analysis and finding an effective solution, with an emphasis on systematicity and the use of concepts known in the field of computer science.
This workshop starts from the description and builds on basic commands to understand how to solve sistematically mazes. In the last stages, i.e. the highest goals in the learning outcomes, the students will be able to understand the abstraction and go over it by changing the mazes themselves2

To the larners it should be made clear that the code they are running is the same as the one that powers the simulation. They should be given access to the original code, in order to copy parts and recognise useful bits

Learners should be exposed to the whole code of the simulation and not just the code in the interface of the game. In that way, even not knowing programming, they can recognise patterns and gain insights that can help them
