Application layer

  • viene esposto un front-end chiamato middlebox che è un router o uno switch, che legge gli header di tutti i livelli e smista le richieste in base ad essi, facendo load balancing, violando i principi della gerarchia.
  • ogni richiesta arriva in broadcast e viene soddisfatta solo da un server in base a qualche principio, es. gli ultimi bit dell'IP sorgente
  • si usa il DNS per restituire una lista di indirizzi IP in rotazione, in modo che ogni client ne usi uno diverso.

L'architettura di Content distribution network è un ulteriore approccio che sovverte l'idea del caching. Il sito stesso distribuisce il contenuto su una serie di server distribuiti geograficamente. Quando un client fa una richiesta CDN, verrà indicato il server più vicino.

Esistono due tecniche:

  • mirroring, in cui il contenuto è copiato su vari nodi, i cui link sono forniti sul sito web
  • DNS redirection, in cui è il name server a fornire un IP diverso per il server a seconda della posizione geografica del client.
